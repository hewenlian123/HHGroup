# 迁移 202604141000：expenses 表增加 source、source_id 与 paid 状态

## 确认是否已执行

在 Supabase 控制台 → **SQL Editor** 中执行：

```sql
SELECT column_name
FROM information_schema.columns
WHERE table_schema = 'public' AND table_name = 'expenses'
  AND column_name IN ('source', 'source_id')
ORDER BY column_name;
```

- 若返回两行（`source`, `source_id`），说明迁移**已执行**。
- 若返回空，说明迁移**未执行**，请按下面方式之一执行。

## 方式一：Supabase 控制台（推荐）

1. 打开 [Supabase Dashboard](https://supabase.com/dashboard) → 选择项目 → **SQL Editor**。
2. 新建 query，粘贴并执行 `202604141000_expenses_source_source_id_paid.sql` 的完整内容。
3. 执行后到 **Project Settings** → **API** → **Reload schema cache** 刷新 schema 缓存。

## 方式二：本地脚本（需直连 URL）

在项目根目录的 `.env.local` 中配置 Supabase 直连数据库 URL：

- `SUPABASE_DATABASE_URL` 或 `DATABASE_URL`
- 取值：Supabase → Project Settings → Database → **Connection string**（URI，建议用 Transaction pooler）

然后执行：

```bash
npx tsx scripts/run-migration-202604141000.ts
```

## 方式三：Supabase CLI

若已 `supabase link` 且迁移历史一致，可执行：

```bash
supabase db push
```

若出现 “Remote migration versions not found in local” 等不一致，可先在控制台用方式一执行上述 SQL，或联系维护者统一迁移历史后再用 CLI。
